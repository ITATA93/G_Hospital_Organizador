# Architectural Decision Records

Este directorio contiene todas las decisiones arquitectónicas importantes del proyecto.

## Estados

- **Proposed**: Propuesta bajo discusión
- **Accepted**: Decisión aceptada e implementada
- **Deprecated**: Ya no se recomienda, pero aún en uso
- **Superseded**: Reemplazada por otra ADR (indicar cuál)

## Cómo Crear un ADR

1. Determina el próximo número (último ADR + 1)

2. Copia la plantilla:
   ```bash
   cp adr-template.md 001-mi-decision.md
   ```

3. Completa todas las secciones:
   - Contexto del problema
   - Decisión tomada
   - Alternativas consideradas
   - Consecuencias (positivas y negativas)

4. Actualiza este README agregando la entrada en la tabla
